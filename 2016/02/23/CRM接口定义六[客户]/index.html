CRM APP接口定义六[客户]

1:获取客户列表

接口说明 :获取客户列表

该接口返回的是指定时间内所添加修改的用户数据

请求地址 /api/v1/customer/list 类型
请求参数 time 指定时间 String
请求参数 type 类型（全部客户，我的客户，我参与的客户，我下级的客户） String
返回值 result 1 操作成功 0 操作失败 int
data json

请求参数 title 职位 String
请求参数 dept 部门 String

请求参数 mail 邮件 String

7:客户查重

手机号唯一标识用户，查询同一手机号下关联的多少用户数据，可能有多条。返回的数据需要有销售人员信息。

请求地址 /api/v1/customer/check/duplicate 类型
请求参数 tel 手机号 String
返回值 result 1 操作成功 0 操作失败 int
data json

例子

8:客户锁定

已手机好为唯一标识，也就是说可能会锁定多条数据。锁定就是分配用户给最开始的用户创建者。

请求地址 /api/v1/customer/lock 类型
请求参数 tel 手机号 String
返回值 result 1 操作成功 0 操作失败 int
data json

例子

9:客户分配

调用此接口前需要先调用查重获取该用户目前有多少销售在使用。分配后和锁定功能相同。

请求地址 /api/v1/customer/distribution 类型
请求参数 tel 手机号 String
请求参数 duid 分发给某个销售，该销售的uid String
返回值 result 1 操作成功 0 操作失败 int
data json
