token没有失效期，跟设备id绑定，除非更换设备，否则不需要重新登录获取token

1:登录接口

接口说明 :用户登录时调用

请求地址 /api/v1/u/login 类型
请求参数 mobile 手机号 String
pwd 密码 String
返回值 result 1 操作成功 0 操作失败 int
data json

例子

2:注册接口

在用户没有经过手机绑定这一步，直接注册是无效的。

接口说明 :用户注册时调用

请求地址 /api/v1/u/reg 类型
请求参数 mobile 手机号 String
pwd 密码 String
返回值 result 1 操作成功 0 操作失败 int
data json
msg String

例子

3:修改密码

接口说明 :用户保存修改的密码时调用

请求地址 /api/v1/u/updatepwd 类型
请求参数 appUserId 当前用户Id String
orginalPwd 原始密码 String
newPwd 新密码 String
返回值 result 1 操作成功 0 操作失败
data json

例子

4:忘记密码

接口说明 :忘记密码，保存新的密码时调用

请求地址 /api/v1/u/forgetpwd 类型
请求参数 mobile 手机号 String
identifyCode 手机短信验证码 String
newPwd 新密码 String
返回值 result 1 操作成功 0 操作失败 int
data json

例子

5:发送短信验证码

接口说明 : 同一个手机号，在120秒之内只能发一次验证码请求
注册（判断是否已经注册过，注册过不发验证码）
忘记密码（一天10次）

请求地址 /api/v1/auth/pushIdentifyCode 类型
请求参数 mobile 手机号 String
请求参数 action 动作（注册，忘记密码） String
返回值 result 1 操作成功 0 操作失败 int
data String

例子

6:验证验证码

接口说明 : 同一个手机号，一个小时只能发30条信息

请求地址 /api/v1/auth/checkIdentifyCode 类型
请求参数 mobile 手机号 String
identifyCode 验证码 String
返回值 result 1 验证成功 2 验证码超时 3 验证码错误 0 操作失败
data json
